# KUB Connect — Руководство пользователя

KUB Connect — десктопный VPN-клиент компании КУБ2Б. Работает на Windows и Linux. Использует протокол VLESS через ядро Xray.

## Первый запуск

При первом запуске приложение откроется без серверов. Нужно добавить хотя бы один сервер перед подключением.

На **Windows** приложение попросит права администратора только если выбрать режим TUN — для режима Proxy достаточно обычного запуска.

---

## Добавление сервера

1. Откройте вкладку **Настройки**.
2. Вставьте ссылку вида `vless://...` в поле ввода.
3. Нажмите **Добавить**.
4. Сервер появится в списке и станет активным.

Чтобы удалить сервер — нажмите иконку удаления рядом с ним в списке.

Кнопка **Пинг** рядом с сервером проверяет его доступность и показывает задержку в миллисекундах (или ошибку). Подключение для этого не требуется.

> Ссылку `vless://...` предоставляет ваш администратор или выдаётся на странице личного кабинета.

---

## Подключение

1. Выберите нужный сервер из выпадающего списка на главном экране.
2. Нажмите **кнопку питания** — она же запускает и останавливает соединение.
3. Статус изменится: **Подключение…** → **Подключено**.

Чтобы отключиться — нажмите кнопку питания ещё раз.

## Режимы работы

Переключается в **Настройки → Подключение**.

### Proxy

- Настраивает системный SOCKS5-прокси на `localhost:10808`.
- Работает без прав администратора.
- Большинство браузеров и приложений используют системный прокси автоматически.
- Трафик некоторых программ может обходить прокси.

### TUN

- Создаёт виртуальный сетевой интерфейс и перехватывает **весь** трафик устройства.
- Требует прав администратора.
- При переключении в TUN без прав администратора появится диалог с предложением перезапустить приложение с правами администратора или вернуться в режим Proxy.

> **Рекомендуется:** для максимальной совместимости используйте TUN.

## Автозапуск и автоподключение

Обе галочки в **Настройках**:

| Настройка | Что делает |
|-----------|-----------|
| **Автозапуск при входе в систему** | Приложение само стартует при входе в ОС. |
| **Автоподключение при старте** | При запуске приложение сразу подключается к активному серверу. |

В режиме **TUN** автоподключение требует прав администратора: при старте появится запрос UAC (Windows), и после подтверждения приложение перезапустится с правами и подключится. Если запрос отклонить — приложение останется запущенным без подключения. В режиме **Proxy** подключение происходит без запроса прав.

> Связка обеих галочек: ОС стартует → приложение поднимается → VPN подключается автоматически.

## Настройки маршрутизации

Блок **Настройки → Что через VPN** определяет, какой трафик идёт через туннель.
Интерфейс контекстный: показываются только те переключатели, которые действуют
в текущем режиме — остальные скрыты, чтобы не сбивать с толку. Под каждым контролом
есть короткое пояснение.

### Что через VPN

- **Только выбранное** — по умолчанию весь трафик идёт напрямую, через VPN — только
  явно выбранное (сейчас это Telegram). Доступен переключатель **Telegram → VPN**.
- **Всё через VPN** — весь трафик идёт через туннель. Локальная сеть (LAN) всегда
  напрямую; IPv6 на время подключения блокируется (защита от утечки). Доступен
  переключатель **RU напрямую** — оставить российские сайты мимо VPN.

| Переключатель | Когда виден | Что делает |
|-----------|-----------|-----------|
| **Telegram → VPN** | режим «Только выбранное» | Направляет трафик Telegram через VPN |
| **RU напрямую** | режим «Всё через VPN» | Российские IP и сайты (geoip:ru) идут напрямую, минуя VPN |
| **Kill switch** | TUN + «Только выбранное» + поддерживаемая ОС (Linux) | Блокирует трафик выбранных адресов при обрыве VPN, чтобы он не утёк напрямую |
| **Mux** | всегда | Мультиплексирование соединений (включайте только если сервер поддерживает) |

> Переключатель сохраняет своё значение, даже когда скрыт: при возврате в режим,
> где он действует, прежняя настройка восстановится.

## Системный трей

При закрытии окна приложение спросит: **свернуть в трей** или **выйти**.

- **Свернуть в трей** — приложение продолжает работать, VPN-соединение не прерывается. Иконка в трее позволяет открыть окно или выйти.
- **Выйти** — приложение завершается, VPN отключается.

## Логи

Вкладка **Логи** показывает события в реальном времени.

Уровень детализации меняется в **Настройки → Уровень логов**:

| Уровень | Описание |
|---------|----------|
| Тихо | Только ошибки |
| Обычный | Предупреждения и ошибки |
| Подробно | Полная отладочная информация |

Кнопка **Очистить** удаляет содержимое лога на экране.

## Обновления

При запуске приложение проверяет наличие новой версии. Если она доступна, сверху появится баннер. Баннер можно закрыть крестиком.

**Windows.** Нажмите **Обновить** в баннере — приложение само скачает установщик новой версии (с индикатором прогресса), запросит права администратора и запустится установщик, а приложение закроется для установки поверх. Если что-то пошло не так — рядом есть ссылка **в браузере** на страницу релиза для ручного скачивания.

**Linux.** Скачайте новую сборку по ссылке **в браузере** и установите вручную.

Установщик при запуске определяет уже установленную версию и предлагает обновление, закрывая запущенное приложение перед установкой.

## Удаление приложения

При удалении (Windows) установщик спросит: **удалить ли настройки и данные** —
список серверов, настройки, geo-базы и драйвер TUN.

- **Да** — все данные приложения стираются полностью.
- **Нет** — данные сохраняются, чтобы повторная установка не сбросила серверы
  и настройки. Это вариант по умолчанию.

Сама программа и ярлыки удаляются в любом случае.

## Решение проблем

### Приложение не подключается

1. Проверьте, что сервер добавлен и выбран.
2. Убедитесь, что ссылка `vless://` действующая — уточните у администратора.
3. Переключите **Уровень логов** на «Подробно» и посмотрите на вкладке **Логи**, какая ошибка выводится.

### В режиме TUN нет кнопки «Запустить от администратора»

TUN-режим на данной ОС не поддерживается. Используйте режим **Proxy**.

### Браузер не использует прокси (режим Proxy)

Некоторые браузеры игнорируют системный прокси. В настройках браузера вручную укажите прокси-сервер: `SOCKS5 localhost:10808`.

### Windows: предупреждение SmartScreen при установке

Приложение не имеет цифровой подписи. Нажмите **Подробнее → Всё равно выполнить**. Это ожидаемое поведение для неподписанного ПО.

### Все сайты открываются медленно после подключения

Попробуйте переключиться с TUN на Proxy или выбрать другой сервер.
