# UI/UX fixes — дизайн

Дата: 2026-07-02

## Контекст

Разбор текущего UI (`frontend/index.html`, `frontend/src/main.ts`, `frontend/src/style.css`) как профессиональный UI/UX-аудит выявил 9 проблем. Иконка power-button уже заменена на inline SVG отдельным изменением (font-glyph `⏻` не рендерился на части систем) — в этот спек не входит.

Ниже — по каждому пункту: проблема, решение, детали реализации.

## 1. Power-button: неразличимые состояния disconnected/error

**Проблема:** `.power.disconnected, .power.error` в `style.css` делят один селектор — идентичный серый цвет. Пользователь не отличит "выключено" от "ошибка подключения" без чтения `status-text`.

**Решение:**
- Разделить селекторы: `.power.disconnected` остаётся серым (`--muted`), `.power.error` получает собственный приглушённо-красный фон (не ярко-`--error`, чтобы не выглядеть как алярм на пустом месте — это просто "не подключено из-за ошибки").
- В error-состоянии SVG-иконка внутри кнопки меняется с power-glyph на восклицательный знак (тот же viewBox 24x24, простой path — круг+точка), чтобы отличие не зависело только от восприятия цвета (доступность для дальтоников).
- Pulse-анимация у connecting/disconnecting не трогается — уже работает.

**Файлы:** `style.css` (новый селектор `.power.error`), `main.ts` (в `render()` — переключение SVG-содержимого кнопки по `st.conn === "error"`), `index.html` (добавить второй `<svg>` с id для error-иконки, скрытый по умолчанию, либо переключать через innerHTML — решение на этапе реализации).

## 2. Settings — плоская стена из 7 секций

**Проблема:** `Серверы → Подключение → Что через VPN → Исключения → Kill switch/Mux → Запуск → Логи` — один длинный скролл без иерархии.

**Решение:** обернуть секции в нативный `<details>`/`<summary>` (без JS). Открыты по умолчанию (`open` атрибут): "Серверы", "Подключение". Свёрнуты по умолчанию: "Что через VPN", "Запуск", "Логи" — но см. пункт 9, где это пересекается с простым/расширенным режимом.

**Файлы:** `index.html` (обернуть существующие `<h2>`+секции в `<details><summary>`), `style.css` (стилизация `summary` под текущий вид `h2`, убрать нативный маркер-треугольник или оставить — решить в реализации по вкусу, стандартный маркер ОК).

## 3. Presets — чекбоксы без визуального акцента

**Решение:** CSS toggle-switch вместо нативного вида checkbox. Разметка не меняется (`<input type="checkbox">` остаётся источником состояния и обработчиком в `main.ts` — ноль изменений в JS), меняется только CSS: `input[type=checkbox]` в `.preset-list label` рендерится как pill-переключатель (`appearance: none` + `::before` кружок + transition). Иконки сервисов НЕ добавляем (лицензии сторонних логотипов, лишний вес) — решено при брейнсторме как дефолт без ответа пользователя.

**Файлы:** `style.css` только.

## 4. Ошибки — статичный текст без внимания

**Проблема:** `#error-line`, `#link-error`, `#routing-error` — обычный `<p>`, легко пропустить, не исчезает сам.

**Решение:** визуально превратить в toast, всплывающий над power-button (top of `.home` / fixed сверху `#app`). Один общий CSS-класс `.toast` применяется к тем же трём существующим элементам (DOM id не меняются — код в `main.ts`, который пишет в `textContent`, не трогаем). Появление: fade-in 150ms при непустом content. Исчезновение: auto-fade через 5s ИЛИ клик по toast. Реализация auto-fade — `setTimeout` в обёртке над установкой `textContent` (маленький helper `setError(el, text)` вместо прямого `.textContent =`).

**Файлы:** `style.css` (`.toast` + анимация), `main.ts` (helper-функция, используется в местах, где сейчас `$("error-line").textContent = ...` и аналоги — таких мест ~8, все меняются на вызов helper'а).

## 5. Нет onboarding при пустом списке серверов

**Решение:** в `refresh()`/`render()` — если `st.servers.length === 0` и это первый рендер с пустым списком (не долбить `setTab` при каждом ре-рендере, если юзер уже сам ушёл на другую вкладку) — переключить на `tab-settings`, `.focus()` на `#link-input`, добавить временный CSS-класс подсветки рамки (`accent`-цвет outline), снимаемый по первому `input`-событию поля.

Флаг "уже показали onboarding" — модуль-level переменная в `main.ts` (не persist), сбрасывается только при перезапуске приложения — ok, onboarding раз за сессию достаточно.

**Файлы:** `main.ts` (`render()` + `wire()` для снятия подсветки), `style.css` (`.onboard-hint` класс на `#link-input`).

## 6. Mux/kill-switch — жаргон без объяснения

**Решение:** добавить `title` атрибут на `<label>` с человеко-понятным текстом:
- Mux: `title="Ускоряет соединение для Telegram — включайте, если сервер настроен без flow-контроля"`.
- Kill switch: уже есть `<span class="hint">`, оставляем как есть — только сверить, что текст понятен (текущий — понятен, не трогаем).

**Файлы:** `index.html` (один атрибут).

## 7. Ping-результат — низкая видимость сигнала

**Решение:** цвет `.ping-result` по значению latency, назначается в момент `apply(text)` в `main.ts` (там, где сейчас пишется `${r.latencyMs} мс`):
- `< 150мс` → `--accent` (зелёный)
- `150–400мс` → `--warn` (жёлтый)
- `> 400мс` или `r.error` → `--error` (красный)

Убрать `opacity: .8` у `.ping-result` в CSS — цвет уже даёт достаточный контраст без затемнения.

**Файлы:** `main.ts` (добавить classList-логику в `apply()`), `style.css` (убрать opacity, добавить color-модификаторы `.ping-result.fast/.slow/.bad` или инлайн через CSS-переменную).

## 8. Close-modal — danger button не выделен визуально

**Решение:** "Выйти" (`#modal-quit`) теряет full-width primary-подобный вид, становится текстовой ссылкой-кнопкой с `border-top` разделителем сверху (визуально отделена от "Свернуть в трей"/"Отмена"). "Свернуть в трей" остаётся единственной full-width `primary` кнопкой — основной, ожидаемый путь.

**Файлы:** `style.css` (`.modal-btn.danger` — новый стиль: `border: none`, `border-top: 1px solid var(--border)`, `border-radius: 0`, уменьшенный `padding`), разметка `index.html` не меняется (порядок кнопок уже подходящий).

## 9. Простой/расширенный режим настроек

**Решение:** toggle (чекбокс или два кнопки-таба) в верхней части `tab-settings`, состояние — `localStorage['ui.advanced']` (булево, не идёт на бэкенд — чисто UI-предпочтение, переживает перезапуск).

Группировка:
- **Простой** (по умолчанию для новых юзеров, т.е. когда `localStorage` пуст): "Серверы", "Подключение" (mode select).
- **Расширенный**: добавляет "Что через VPN" (presets/textarea/RU-direct/exceptions), "Kill switch", "Mux", "Уровень логов". "Запуск" (автозапуск/автоподключение) остаётся видимым в обоих — не routing-специфично, нужно всем.

Реализация — CSS-класс на `#tab-settings` (`.simple-mode`), скрывающий `<details>`-блоки с `data-advanced` атрибутом через `[data-advanced].hidden` или `:not(.simple-mode) [data-advanced]` — деталь на этапе плана.

**Файлы:** `index.html` (toggle-контрол + `data-advanced` атрибуты на секциях), `main.ts` (чтение/запись `localStorage`, применение класса), `style.css` (видимость по классу).

## Не входит в этот спек

- Замена power-glyph на SVG — сделано ранее, отдельный коммит.
- Иконки сервисов для presets — отклонено (лицензии/вес).
- Полный редизайн routing UI в карточки — не выбрано пользователем, toggle-switch чекбоксов достаточно.
