# Changelog

## v1.3.0 — 2026-07-02

### Новое
- Пресет **Claude (Anthropic)** в списке сервисов для whitelist-маршрутизации.
- Простой/расширенный режим настроек — переключатель наверху Настроек, скрывает
  редко используемые секции (маршрутизация, логи) для новых пользователей.

### Улучшения
- Telegram перенесён в общую сетку пресетов сервисов (как единый переключатель),
  по-прежнему включён по умолчанию.
- Настройки разбиты на сворачиваемые секции вместо одной длинной страницы.
- Пресеты сервисов и переключатель режима теперь выглядят как тумблеры.
- Ошибки показываются всплывающими уведомлениями с автоскрытием вместо
  статичного текста.
- При пустом списке серверов приложение сразу открывает Настройки и подсвечивает
  поле для ссылки.
- Результат проверки пинга сервера окрашен по задержке (зелёный/жёлтый/красный).
- Подсказка при наведении поясняет, что делает переключатель Mux.

### Исправления
- Индикатор состояния подключения (кнопка питания) больше не выглядит одинаково
  при отключении и при ошибке подключения.
- Кнопка «Выйти» в диалоге закрытия визуально отделена от основного действия
  «Свернуть в трей», чтобы её было сложнее нажать по ошибке.

## v1.2.1 — 2026-06-22

### Исправления
- Авто-обновление выбирало установщик случайно из двух (Windows 10 и Windows 7),
  появившихся в v1.2.0 — на Windows 10/11 мог скачаться Win7-установщик. Теперь
  установщик выбирается по версии ОС: Win10+ → обычный, Win7/8 → windows7.

## v1.2.0 — 2026-06-22

### Новое
- Отдельная сборка для **Windows 7 SP1 / Server 2008 R2** —
  `kub-connect-vX.Y.Z-windows7-amd64-installer.exe`. Собирается патченным
  toolchain XTLS go-win7 (официальный Go 1.21+ на Win7 падает при старте).
  Обычный установщик по-прежнему требует Windows 10+. Требования к Win7:
  обновления KB2533623, KB4474419 и WebView2 Runtime (см. docs/INSTALL.md).

## v1.1.3 — 2026-06-16

### Исправления
- Повторный запуск приложения больше не открывает второй экземпляр: новый
  запуск разворачивает уже работающее окно (в том числе из трея) и завершается.
- TUN-режим: исправлено подключение на системах с сетевым адаптером, названным
  не латиницей (например «Беспроводная сеть»). Раньше маршруты не применялись с
  ошибкой netsh, потому что имя адаптера портилось из-за кодировки вывода
  PowerShell. Теперь используется числовой индекс интерфейса.

## v1.1.2 — 2026-06-15

### Улучшения
- При удалении приложения деинсталлятор спрашивает, удалить ли настройки и
  данные (сервера, настройки, geo-базы, драйвер TUN). По умолчанию данные
  сохраняются — чтобы переустановка не сбрасывала список серверов и настройки.

## v1.1.1 — 2026-06-15

### Улучшения
- Настройки маршрутизации стали понятнее: контролы, которые не действуют в
  текущем режиме, теперь скрыты, а не показаны вхолостую. «RU напрямую» видно
  только в режиме «Всё через VPN», Kill switch — только там, где он реально
  работает (TUN + поддерживаемая ОС). Заголовки и пункты переименованы
  по-человечески, под каждым контролом добавлены короткие пояснения.

## v1.1.0 — 2026-06-15

### Возможности
- Обновление из приложения (Windows): по кнопке «Обновить» в баннере приложение
  само скачивает установщик новой версии с индикатором прогресса, запускает его
  (с запросом прав администратора) и закрывается для установки поверх. Ссылка на
  скачивание в браузере осталась как запасной вариант.
- Установщик определяет уже установленную версию и предлагает обновление, закрывая
  запущенное приложение перед установкой.

## v1.0.14 — 2026-06-11

### Возможности
- Новый режим маршрутизации «Всё через VPN» (full tunnel) для Proxy и TUN. LAN остаётся
  напрямую, IPv6 блокируется на время подключения, RU-Direct можно комбинировать.

### Исправления
- Автоподключение при старте в режиме TUN теперь запрашивает права администратора
  (раньше тихо не срабатывало без прав).

## v1.0.13 — 2026-06-11

### Сборка и релизы
- Релизы публикуются через GitHub Actions на self-hosted раннере.
- В релиз идёт Windows NSIS-installer (а не голый .exe).

### Исправления
- Проверка обновлений сравнивает версии по semver, а не на неравенство
  (билд впереди релиза больше не показывает ложный баннер).
- Frontend: устранена HTML-инъекция в списке серверов, потеря индикатора
  пинга при ре-рендере, безграничный рост логов и двойное добавление сервера.

## v1.0.1 — 2026-06-09

### Исправления
- CI: метки раннера приведены к GitVerse (`img-ubuntu-latest`).

## v1.0.0 — 2026-06-09

Первый публичный релиз под брендом KUB Connect.

### Возможности
- VLESS+Xray клиент с whitelist-маршрутизацией (Telegram + кастомные IP/домены).
- **Windows:** режимы Proxy и TUN.
- Встроенные geo-базы (geoip/geosite) — отдельная установка не нужна.
- GUI на Wails: управление серверами, выбор режима, уровень логов.
- Автопроверка обновлений — баннер при выходе новой версии.

### Известные ограничения
- Kill switch не реализован.
- Windows-бинарь не подписан (SmartScreen warning).

## v0.1.0 — 2026-06-08

Первый публичный релиз.

### Возможности
- VLESS+Xray клиент с whitelist-маршрутизацией (Telegram + кастомные IP/домены).
- **Windows:** режимы Proxy и TUN.
- **macOS:** режим Proxy.
- Встроенные geo-базы (geoip/geosite) — отдельная установка не нужна.
- GUI на Wails: управление серверами, выбор режима, уровень логов, логи.

### Известные ограничения
- Kill switch не реализован.
- macOS TUN не поддерживается.
- Windows-бинарь не подписан (SmartScreen warning).
